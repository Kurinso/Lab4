Задания 1-5: Статический класс ProblemSolver
Задание 1: Замена первого вхождения списка
Реализация: Метод ReplaceFirstOccurrence<T>()

Алгоритм поиска первого вхождения списка L1 в список L

При нахождении заменяет элементы L1 на элементы L2

Возвращает новый список с выполненной заменой

Сложность O(n*m), где n и m - длины списков

Тестирование:

Исходный список: {1, 2, 3, 4, 5, 3, 4, 5, -1, -5}

Заменяемый: {3, 4, 5, -1}

Замена: {9, 9, 9}

Результат: {1, 2, 9, 9, 9, 3, 4, 5, -5}

Задание 2: Сортировка списка
Реализация: Метод SortList<T>() where T : IComparable<T>

Использует встроенный метод List.Sort()

Возвращает новый отсортированный список

Поддерживает любой тип, реализующий IComparable

Тестирование:

Исходный: {5, 2, 8, 1, 9, -1, 0, 10012312, -14124}

Результат: {-14124, -1, 0, 1, 2, 5, 8, 9, 10012312}

Задание 3: Анализ компьютерных игр
Реализация: Метод AnalyzeGames()

Анализирует, в какие игры играют студенты

Возвращает три списка:

Игры, в которые играют все студенты

Игры, в которые играют некоторые студенты

Игры, в которые не играет никто

Тестирование:

Все игры: {"CS:GO", "Dota 2", "Minecraft", "Fortnite", "qwer"}

Студенты играют в различные комбинации

Результаты соответствуют ожиданиям

Задание 4: Глухие согласные буквы
Реализация: Метод FindVoicelessConsonants()

Читает текст из файла

Находит глухие согласные, которые не входят хотя бы в одно слово

Возвращает отсортированный список согласных

Глухие согласные: п, ф, к, т, ш, с, х, ц, ч, щ

Особенности реализации:

Учитывает русскую кодировку (UTF-8)

Игнорирует знаки препинания

Приводит текст к нижнему регистру

Задание 5: Олимпиада по информатике
Реализация:

Метод CreateParticipantsFile() - создание файла с данными

Метод FindWinnersFromFile() - чтение файла и определение победителей

Приватный класс Participant для хранения данных об участниках

Метод ParseParticipant() для разбора строки

Логика определения победителей:

Суммируются баллы за три задания

Находится максимальная сумма

Выбираются все участники с этой суммой

Тестирование:

Создан файл с 5 участниками

Два участника набрали максимальный балл (75)

Вывод: "Сидоров Петр, Кузнецов Алексей"

Задания 6-7: Класс Money
Структура класса:
Поля:

rubles (uint) - количество рублей

kopeks (byte) - количество копеек (0-99)

Конструкторы:

По умолчанию (0 руб. 0 коп.)

С параметрами (рубли, копейки)

Из общего количества копеек

Основные методы:

AddKopeks(uint) - добавление копеек (задание 6)

Normalize() - нормализация (копейки → рубли)

ToString() - строковое представление

Перегруженные операторы (задание 7):
Унарные операции:

++ - добавление одной копейки

-- - вычитание одной копейки

Операции приведения типа:

explicit operator uint - получение рублей (копейки отбрасываются)

implicit operator double - копейки в рублях (без целой части)

Бинарные операции:

+ между Money и uint (коммутативная)

- между Money и uint

+ между двумя Money

- между двумя Money

Обработка ошибок:
Проверка диапазона копеек (0-99)

Проверка на отрицательный результат при вычитании

Защита от вычитания из нуля

Тестирование:
Протестированы все сценарии:

Корректное создание объектов

Добавление копеек с переполнением

Все перегруженные операторы

Ошибочные ситуации с обработкой исключений
